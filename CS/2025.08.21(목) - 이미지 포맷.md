## 이미지가 만들어지는 방식

1. 비트맵 (Raster)
2. 벡터 (Vector)

이외에 많이 보이는 PNG, JPEG, WebP, SVG, PDF는 **이 두 방식 중 어디에 속하는 파일 형식인지의 차이**

## 1. 비트맵 vs 벡터

### 1-1. 비트맵 (Raster)

---

그림을 **픽셀(점)** 로 저장함

```markdown
■■■■■■
■■■■■■
■■■■■■
```

이렇게 생겨서 사진을 확대하면 점으로 보이면서 깨지게 되는 것

**특징**

- 픽셀(점)으로 구성
- 확대하면 깨짐
- 사진이나 복잡한 그림에 사용됨
- 대표적으로 PNG, JPEG, WebP가 있음

### 1-2. 벡터 (Vector)

---

그림을 **수학공식**으로 저장함

```markdown
원 중심 (100,100)
반지름 50
색상 빨강
```

이런식으로 저장해서, 확대해도 깨지지 않음 

<aside>

### 벡터가 확대해도 깨지지 않는 이유

---

벡터는 비트맵과 다르게 픽셀이 저장되어있지 않음. 다만, **그리는 방법**이 저장된 것임

비트맵은 이미 그려져있으니 확대하면 그 점들이 자세하게 보여지며 깨지지만, 벡터는 수학 공식이 있으니, 확대하면 그 화면 기준으로 수학 공식이 적용되어서 **계산해서 새로 그리게 됨**

즉, **“확대 → 다시 계산 → 다시 그리기”** 가 반복되는 것이라서 깨지지 않음

**차이점**

- 비트맵: 이미 그려진 그림을 확대
- 벡터: 확대할 때 수학 공식에 따라 새로 계산해서 다시그림
</aside>

**특징**

- 수학공식으로 구성
- 확대해도 깨지지 않음
- 아이콘이나 로고처럼 깨지지 않길 원하는 곳에 사용됨
- 대표적으로 SVG, PDF가 있음

<br>

## 2. PNG

PNG는 가장 일반적으로 쓰이는 **비트맵 이미지**로 무손실 압축과 투명 배경을 지원하지만, 파일 크기는 JPEG보다 크다.

**언제 쓰나?**

- 투명 배경이 필요한 이미지 → 예: 배경 없는 로고, 스티커, 버튼 위에 올라가는 오버레이 이미지
- 선명도가 중요한 UI 에셋 → 예: 앱 내 일러스트, 온보딩 이미지처럼 압축 손실이 생기면 안 되는 경우

<br>

## 3. JPEG(JPG)

JPEG도 **비트맵 이미지**로, 손실 압축 방식이며 투명 배경은 지원하지 않는다. 다만 PNG보다 파일 크기는 작다.

**언제 쓰나?**

- 색상이 풍부하고 복잡한 사진 → 예: 음식 사진, 풍경 사진처럼 수백만 가지 색이 섞인 이미지
- 투명도가 필요 없고 용량을 줄여야 할 때 → 예: 서버에 올리는 프로필 사진, 피드에 표시되는 썸네일

<br>

## 4. WebP

WebP는 **구글이 만든 최신 비트맵 포맷**이다. 압축이 매우 잘되서 용량이 매우 작아 PNG/JPEG보다 용량이 작다. 투명 배경 또한 지원한다.

다만 네이티브에서는 여전히 PNG/JPEG가 더 많이 사용됨

**언제 쓰나?**

- 웹 서비스에서 이미지 로딩 속도를 최적화할 때 → 예: 쇼핑몰 상품 이미지, 뉴스 기사 썸네일처럼 수십~수백 장이 한 페이지에 뜨는 경우
- PNG/JPEG 대비 동일 품질에서 파일 크기를 30~50% 줄이고 싶을 때 (단, iOS 네이티브 앱보다는 웹뷰·서버 응답 이미지에 적합)

<br>

## 5. SVG



SVG는 **벡터 이미지 포맷**으로, 확대시 깨지지 않으며 XML기반이다.

웹에서는 많이 사용되지만 iOS에서는 SVG를 직접 지원하지 않아 보통 PDF 벡터 에셋으로 변환해서 사용한다.

아래처럼 저장되는 방식이다.

```xml
<circle cx="100" cy="100" r="50" fill="red"/>
```

**언제 쓰나?**

- 다양한 해상도(1x, 2x, 3x)에 대응해야 하는 단순한 아이콘·일러스트 → 예: 탭바 아이콘, 버튼 아이콘처럼 선과 도형으로 이루어진 심플한 그래픽에 사용됨

<br>

## 6. PDF

PDF는 조금 특이하다. **벡터도 되고 비트맵도 된다.**

그래서 iOS에서는 깨지지 않는 **벡터 이미지로 많이 사용**하며, 커스텀 벡터 에셋에서 Apple이 오랫동안 권장해온 방식이다. (현재는 SF Symbols을 우선 권장)

iOS에서는 `Single Scale PDF` 이렇게 사용하면, `1x`, `2x`, `3x` 이미지를 자동으로 만들어준다.

**언제 쓰나?**

- Xcode Asset에 커스텀 벡터 아이콘을 추가할 때 → 예: SF Symbols로 대체가 안 되는 앱 고유의 탭바 아이콘, 버튼 아이콘
- "Preserve Vector Data" 옵션을 켜면 어떤 크기로 렌더링해도 깨지지 않음 → 예: 스플래시 화면 로고처럼 기기마다 다양한 크기로 표시되는 경우
