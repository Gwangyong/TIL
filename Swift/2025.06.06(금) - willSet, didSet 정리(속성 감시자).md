## willSet, didSet이란?

willSet, didSet은 Swift에서 **프로퍼티의 값이 변경될 때 자동으로 실행**되는 코드 블록으로, 이러한 기능을 **속성 감시자(Property Observers)**라고 부른다.

## **📌 언제 사용될까**

- 값이 **변경되었을 때 특정 로직을 실행하고 싶을 때**
- 예: UI 업데이트, 값 변경 로그 출력, 의존성 있는 값 같이 업데이트 등

**작동 방식**

- **`willSet`**: 값이 변경되기 **직전**에 호출됨 (`newValue`)
    - **`newValue`**: 곧 설정될 새로운 값
- **`didSet`**: 값이 변경된 **직후**에 호출됨 (`oldValue`)
    - **`oldValue`**: 변경되기 전의 기존 값

기본 제공은 `newValue`, `oldValue`이며, `willSet(newV)`, `didSet(oldV)` 처럼 원하는 이름으로 변경할 수 있다. 괄호를 생략할 시에는 기본 이름이 자동으로 사용된다.

> **기본 형태**
> 

```swift
var property: Type {
    willSet {
        // 이제 곧 바뀔 값 사용 가능 → newValue
    }
    didSet {
        // 이전 값 사용 가능 → oldValue
    }
}
```

> **사용 예시**
> 

```swift
struct Robot {
    var name: String = "Robot" {
        willSet(newName) { // newValue -> newName
            print("[willSet] 현재 값: \(name)")
            print("[willSet] 곧 바뀔 값: \(newName)")
        }
        didSet(oldName) { // oldValue -> oldName
            print("[didSet] 이전 값: \(oldName)")
            print("[didSet] 변경된 현재 값: \(name)")
        }
    }
}

var robot = Robot()
robot.name = "test"
```

> **출력**
> 

```swift
[willSet] 현재 값: Robot
[willSet] 곧 바뀔 값: test
[didSet] 이전 값: Robot
[didSet] 변경된 현재 값: test
```

## **내부 동작**

- Swift는 `willSet`, `didSet`  호출을 위해 변경 전 값을 내부적으로 **자동 저장**해둔다.
- 개발자가 따로 관리하지 않아도 `newValue` / `oldValue`로 값이 전달됨
