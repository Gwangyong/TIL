오늘은 Swift의 주석에 대해 정리해보려 한다.

## `//` 한 줄 주석

가장 기본적인 주석이다. 코드의 옆이나 위에 간단한 설명을 덧붙일 때 사용한다.

```swift
let name = "Mori" // 사용자 이름
```

**📌 사용 시점**

- 간단한 변수, 함수 설명
- 디버깅용 코드 비활성화
- TODO/FIXME 없이 간단한 작업 메모할때 사용

## `/* ... */` 여러 줄 주석

여러 줄에 걸친 주석을 작성할 때 사용한다.

```swift
/*
 이 함수는 사용자의 이름을 출력합니다.
 내부적으로는 print() 함수를 사용합니다.
*/

func greet() {
	print("Hello")
}
```

**📌 사용 시점**

- 긴 설명이 필요할 때
- 코드 블록 전체를 주석 처리하고 싶을 때
- 중첩 주석도 가능 (/* ... /* ... */ ... */)

> 중첩 주석 사용 예시
> 

```swift
/*
 이 함수는 사용자 정보를 출력하는 함수입니다.
 
 내부적으로 다음과 같은 처리를 합니다:
 
 /* 
 1. 이름이 nil인지 확인합니다
 2. nil이면 기본값을 설정합니다
 */
 
 그리고 최종적으로 print()를 호출합니다.
*/

func printUserInfo(name: String?) {
    let displayName = name ?? "사용자"
    print(displayName)
}
```

- Swift에서는 `/* */` 스타일의 여러 줄 주석 안에, 또 다른 `/* */` 주석을 중첩해서 사용할 수 있다.
- 다른 언어(C, Java 등)에서는 안되는 경우도 있지만, Swift는 이를 지원한다고 한다.

**📌 이런 상황에서 유용해요**

- 문서형 주석 안에 리스트/단계별 설명을 주석으로 추가하고 싶을 때
- 코드 블록을 통째로 주석 처리하려는데, 안에 이미 주석이 있을 경우
- 임시 비활성화할 코드가 많은데, 그 코드에 이미 `/* */` 주석이 포함되어 있을 때

## `// MARK:`  코드 네비게이션용 주석

`// MARK:`는 Xcode 내에서 코드 네비게이터에서 섹션을 구분하는데 사용된다.

뒤에 `-`를 붙이면 시각적으로도 구분선이 생기게된다.

- `// MARK:`  → 텍스트만 보임
- `// MARK: -`  → **구분선 + 텍스트**

```swift
// MARK: - Life Cycle
override func viewDidLoad() {
    super.viewDidLoad()
}
```

**📌 사용 시점**

- 기능별 섹션 구분
- 파일이 길어질 때 코드 탐색을 쉽게 하기 위해
- `// MARK: -`, `// TODO:`, `// FIXME:` 등의 형식도 가능

### `// TODO:`, `// FIXME:` 작업 추적 주석

---

Swift/Xcode는 `TODO`나 `FIXME` 같은 키워드를 자동으로 인식하여 네비게이터에 표시해준다.

```swift
// TODO: 에러 처리 로직 추가 예정
// FIXME: 강제 언래핑 수정 필요
```

**📌 사용 시점**

- 추후 작업 예정 항목 기록
- 나중에 고쳐야 할 코드 표시

## `///` 문서화 주석 (Documentation Comments)

이 주석에 대해 공부하기 시작한 이유가 이 문서화 주석 때문이다.

있는지도 모르고 `//`를 치려다가 하나를 더 쳐서 `///`를 쳤는데, 모양이 조금 달라지길레 “이런 주석도 있나..?” 하고 우선은 프로젝트 하느라 `//`랑 별개로 주석을 달기위해 그냥 사용하다가 이번에 정리하게 되었다.

`///` 는 Swift에서 **공식 문서화 주석**을 작성할 때 사용하는 문법이다. 이 주석은 **Xcode의 Quick Help(Option + 클릭)**에서도 표시된다.

```swift
/// 사용자 정보를 출력하는 함수입니다.
/// - Parameter name: 사용자 이름
func printName(_ name: String) {
	print(name)
}
```

지금 예시로 쓴 것은 `- Parameter` 태그인데, 함수의 매개변수를 설명할 때 사용한다.

이 외에도 `- Throws`,  `- Note`, `- SeeAlso`, `- Version` 등 역할에 따라 여러 태그가 있다.

> 예시로 몇가지 태그를 함께 사용해봤다.

```swift
/// 사용자 정보를 출력합니다.
/// - Parameter name: 사용자 이름
/// - Returns: 출력된 문자열
/// - Warning: 이름이 비어있으면 crash 날 수 있습니다.
/// - SeeAlso: validateName(_:)
func printName(_ name: String) -> String {
    return "Hello, \(name)"
}
```

- 태그들 간단히 정리
    
    
    | 태그 | 설명 | 사용 예시 |
    | --- | --- | --- |
    | **- Parameter** | 함수의 매개변수 설명 | /// - Parameter name: 사용자 이름 |
    | **- Returns** | 반환값 설명 | /// - Returns: 유저 정보 문자열 |
    | **- Throws** | 에러 발생 설명 | /// - Throws: 유효하지 않은 입력일 경우 |
    | **- SeeAlso** | 관련 문서/함수 등 참조 연결 | /// - SeeAlso: printUserInfo(_:) |
    | **- Note** | 특별히 주의할 점 표시 | /// - Note: 비동기 작업이므로 주의하세요 |
    | **- Warning** | 위험하거나 주의가 필요한 부분 강조 | /// - Warning: 이 함수는 메인 쓰레드에서만 호출하세요 |
    | **- Attention** | Warning보다 더 강조한 경고 | /// - Attention: 데이터 손실 위험이 있습니다 |
    | **- Important** | 핵심 포인트 강조 | /// - Important: 로그인 후에만 사용하세요 |
    | **- Author** | 작성자 정보 | /// - Author: Mori |
    | **- Version** | 버전 정보 | /// - Version: 1.0.0 |

**📌 사용 시점**

- 공개 함수나 클래스 설명
- Swift Doc(문서화) 자동 생성 시
- 팀 협업 시 코드 이해를 돕기 위해
