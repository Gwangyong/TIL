## **Auto Layout에서의 “우선순위”란?**

Auto Layout은 뷰의 크기를 계산할 때 이렇게 묻는다.

> 💬 “이 뷰는 **얼마나 본인의 크기를 지키고 싶어하나요?”**
> 

이 질문에 대한 답이 바로 **우선순위(Priority)** 이다.

<br>

## 우선순위의 두 가지 종류

뷰가 intrinsicContentSize(자기 고유 크기)를 기준으로, 얼마나 **늘어나기 / 줄어들기 싫어하는지**를 나타내는 값이다.

#### 1. Content Hugging Priority

> “나는 내 콘텐츠 크기만큼만 있고 싶어. 더 넓어지기 싫어.”
- 값이 클 수록, **늘어나지 않으려 함**
- 남는 공간이 있어도 콘텐츠 크기 이상으로 잘 안 커짐

#### 2. ContentCompression Resistance Priority

> “나는 지금보다 작아지기 싫어.”
- 값이 클수록, **줄어들지 않으려 함**
- 콘텐츠 크기 이상으로 남는 공간이 있어도 줄어들지 않음

| **우선순위** | **의미** |
| --- | --- |
| **Content Hugging Priority** | “내 콘텐츠 크기 딱 맞게 있을래. **지금보다 더 넓어지기 싫어.**” |
| **Content Compression Resistance Priority** | “누가 나를 작게 만들려고 해도, **난 지금보다 작아지기 싫어.**” |

<br>

## **왜 우선순위가 필요할까?**

예를 들어, 이런 상황을 보자.

```swift
[ "Author" ][     J.K Rowling     ]
```

"Author"는 고정 텍스트고, "J.K Rowling"은 길이가 바뀔 수 있는 값이라 가정하자.

여기서 만약, **두 라벨 사이에 공간이 부족하면? 누굴 줄여야할까?**

이럴 때 **우선순위(priority)** 를 사용하여 결정하는 것이다.

> 예시 코드

```swift
let authorTitleLabel = UILabel() // Author
let authorContentLabel = UILabel() // 저자(J.K Rowling)

// 고정 텍스트 (HuggingPriority)
authorTitleLabel.setContentHuggingPriority(.required, for: .horizontal)
authorTitleLabel.setContentCompressionResistancePriority(.required, for: .horizontal)

// 가변 텍스트 (Content Hugging Priority)
authorContentLabel.setContentHuggingPriority(.defaultLow, for: .horizontal)
authorContentLabel.setContentCompressionResistancePriority(.defaultLow, for: .horizontal)
```

<br>

## **각 설정 의미**

#### **1. authorTitleLabel (“Author”)**

- `Hugging: 1000` → “내 크기만큼만 있을래, 더 넓어지기 싫어”
- `CompressionResistance: 1000` → “작아지고 싶지 않아, 글자 안 잘리게 해줘!”

즉, **“가능한 한 자기 크기 유지.”** 고정 텍스트이므로 잘리면 안됨

#### **2. authorValueLabel (“J.K Rowling”)**

- `Hugging: 250` → “내가 좀 넓어져도 괜찮아”
- `CompressionResistance: 250` → “필요하면 작아져도 됨”

즉, “**공간이 모자라면 이 라벨이 넓이 조절을 감수함.”**

이건 사용자 입력 값처럼 **가변적인 정보**니까 줄어들어도 어색하지 않음.

<br>

## 우선순위 값 범위

| 상수 | 값 |
| --- | --- |
| .required | 1000 |
| .defaultHigh | 750 |
| .defaultLow | 250 |
- 범위: 1 ~ 1000
- 숫자가 클 수록 우선순위가 더 강함
- 필요하면 UILayoutPriority(900) 처럼 값을 직접 지정 가능

<br>

## **우선순위를 안주면?**

각 뷰에는 **기본 우선순위 값이 이미 존재**한다. 그렇기에 값을 따로 지정해주지 않게되면, Auto Layout이 그 값을 기준으로 계산하게 된다.

하지만 우선순위가 중요한데 명시해주지 않게되면, 원하지 않는 쪽이 줄어들 수 있게 된다.그래서 **명확하게 값을 지정**해주는것이 좋다.

<br>

## 정리

- 줄어들지 않게 하고 싶다면 → `Compression Resistance Priority`를 높인다.
- 늘어나지 않게 하고 싶다면 → `Hugging Priority`를 높인다.
