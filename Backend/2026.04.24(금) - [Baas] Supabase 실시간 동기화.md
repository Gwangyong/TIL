## 실시간 동기화

Supabase는 **Channels**라는 개념으로 실시간 통신을 처리한다.
채녈 = 여러 클라이언트가 연결되는 **“방”** 같은 개념

실시간 기능은 `Broadcast`, `Postgres Changes`, `Presence` 이렇게 3가지가 있음

<br>

### Broadcast (브로드캐스트)

---

**“클라이언트 → 클라이언트” 직접 메시지 전달**

- **DB를 거치지 않음**
- 보낸 메시지가 같은 채널에 있는 다른 클라이언트들한테 바로 전달됨
- 빠름 (DB 저장 없음)

#### 언제쓸까?

- 채팅 메시지 (굳이 저장 안해도 되는 것)
    - 인스타 DM에 채팅보내고 있다는 `...` 표시 나오는 것
- 마우스 커서 위치 공유
- 실시간 게임 상태

```markdown
클라이언트 A -> [Supabase 서버] -> 클라이언트 B, C, C
							(DB 저장 없음)
```

<br>

### Postgres Changes

---

**“DB 변경사항 → 클라이언트” 알림**

- DB(PostgreSQL)에 **데이터가 바뀔 때 자동**으로 클라이언트에 알려줌
- INSERT / UPDATE / DELETE 감지
- DB를 거침 (Broadcast에 비해 약간 더 느림)

#### 언제쓸까?

- 주문 상태 바뀌면 화면 자동 업데이트
- 다른 사람이 문서 수정하면 내 화면에도 반영
- 재고 변경 실시간 반영

```markdown
누군가 DB에 데이터를 변경함
-> [Supabase 서버가 변경을 감지]
-> 구독 중인 클라이언트들에게 알림
```

<br>

### Presence

---

**“지금 누가 채널에 접속해있는지” 상태 공유**

- 유저가 접속/퇴장하면 같은 채널의 모든 클라이언트가 알 수 있음

#### 언제쓸까?

- 구글 독스 “지금 3명이 보는 중” 표시
- 채팅앱 온라인/오프라인 표시
- 협업툴 접속자 표시 등

```markdown
유저 A가 채널에 접속/퇴장함
-> [Supabase 서버가 감지]
-> 같은 채널의 B, C, D에게 상태 전달
```

<br>

## 비교

|  | Broadcast | Postgres Changes | Presence |
| --- | --- | --- | --- |
| 데이터 저장 | ❌ 안 함 | ✅ DB에 저장됨 | ❌ 안 함 |
| 출발지 | 클라이언트 | DB 변경 | 클라이언트 (접속/퇴장) |
| 속도 | 빠름 | 약간 느림 | 빠름 |
| 용도 | 임시 이벤트 | 데이터 동기화 | 접속자 상태 공유 |

- 한 줄 요약
    - Broadcast: “야 이거 봐!” (저장 없이 그냥 외쳐서 전달)
    - Postgres Changes = “DB 바뀌었으니, 너도 업데이트 해” (DB 변경 감지)
    - Presence: “지금 나 여기 있어!” (접속자 상태를 서로 공유)
