# 2025.09.02(화) - [Podoit] Font 숫자 너비에 따른 View 크기 변화 문제 해결

> **최종 수정일: 2026.03.15 (일)**
> 

오늘도 어김없이 문제가 발생했다.

바로 숫자가 바뀔 때마다 Label의 **너비가 달라져서 View 크기가 흔들리는 문제**다.

![비례글꼴 상태](https://velog.velcdn.com/images/key4168/post/7d46290e-2019-4fbe-a53f-2b4b2c3d268f/image.gif)


원인은 숫자가 **비례 글꼴(proportional font)** 로 렌더링되어 있기 때문이었다. 

비례 글꼴에서는 문자마다 폭이 다르게 설계되어있어, “1”은 좁고 “0”은 넓다. 그래서 값이 바뀔 때마다 Label의 너비가 달라져서 UI가 흔들리게 되는 것이었다.

아래 블로그 보니, monospaced 글꼴을 사용해 폭을 고정하거나, 숫자만 고정폭으로 렌더링 하는 방법이 있었다.

> 참고 블로그
> 
- [SwiftUI - monospaced를 이용한 Text 고정폭 설정하기](https://green1229.tistory.com/390)

---

아래는 애플이 제공하는 **시스템 폰트**를 사용할 시, 기본으로 제공하는 방식이다. (UIKit 기준)

```swift
let font = UIFont.monospacedDigitSystemFont(ofSize: 14, weight: .medium)
goalTimeLabel.font = font
```

다만.. 이번 프로젝트는 **커스텀 폰트**를 사용하기에 위의 방법은 쓸 수 없었다.

그래서 `UIFontDescriptor`를 이용해 직접 구현했다.

> 커스텀 폰트 확장
> 

```swift
extension UIFont {
  // 현재 폰트의 굵기나 크기는 유지. 숫자만 고정폭(monospaced)으로 렌더링
  func monospacedDigits() -> UIFont {
    let features: [[UIFontDescriptor.FeatureKey: Any]] = [
      [
        .featureIdentifier: kNumberSpacingType, // 숫자 간격 기능 그룹
        .typeIdentifier: kMonospacedNumbersSelector // 위 그룹에서 고정폭 옵션 선택
      ]
    ]
    // 기존 fontDescriptor에 새 feature 설정을 병합한 새로운 descriptor(desc) 생성
    let desc = fontDescriptor.addingAttributes([.featureSettings: features])
    return UIFont(descriptor: desc, size: pointSize)
  }
}
```

- 위와같이 `monospacedDigits()`를 직접 구현하였다.

커스텀 폰트 내부에는 여러 `OpenType feature` 옵션이 존재한다. (디자이너가 글꼴에 추가한 고급 타이포그래피 옵션으로, 해당 폰트의 세부 설정이라고 보면 된다.) 이 확장은 그중 숫자 간격 설정만 고정폭으로 변경하는 것이다.

- `UIFontDescriptor`는 폰트의 이름/크기/굵기/feature 설정 등 폰트를 구성하는 **속성 전체를 담은 명세 객체**다. 폰트 설계도? 라고 보면 될 것 같다.
- `.featureIdentifier` / `.typeIdentifier`는 feature 딕셔너리의 키(key)로,
    - 어떤 **기능 그룹**을 수정할지 (`kNumberSpacingType` → 숫자 간격)
    - 위에서 선택한 그룹에서 어떤 **옵션**을 선택할지 (`kMonospacedNumbersSelector` → 고정폭)를 각각 지정한다.
- `addingAttributes(_: )`는 기존 `fontDescriptor`의 속성에 새 속성을 **병합(merge)**한다.
    - **기존 속성은 그대로 유지되고, 같은 key가 있을 때만 새 값으로 대체**된다.
    - 결과적으로 원래 폰트의 규칙 위에 내가 지정한 feature만 추가된 새로운 descriptor가 만들어지는 것이다.
- `pointSize`: UIFont가 원래 가지고 있던 크기(폰트 크기를 유지하기 위해서 사용)

주의할점은, 커스텀 폰트가 `kNumberSpacingType` feature를 내장하고 있을때만 적용이 된다고 한다. 그러니 폰트 스펙을 미리 확인해서 지원하는지 확인해야한다.

> 사용
> 

```swift
// 원래 사용하던 방식
$0.font = Typography.font(for: .labelMd(weight: .medium))

// 위의 코드 추가 후 수정한 방식
$0.font = Typography.font(for: .labelMd(weight: .medium)).monospacedDigits()
```

이렇게 하면 숫자가 항상 고정폭으로 출력되어, 타이머같이 숫자가 변하는 UI에서 View크기가 흔들리지 않게 된다!
