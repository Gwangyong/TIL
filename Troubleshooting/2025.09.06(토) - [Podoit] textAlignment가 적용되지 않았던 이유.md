## 문제 발생!

아래 이미지처럼, 커스텀 알럿에서 Label들이 중앙 정렬이 되지 않는 문제가 있었다. 

(message는 다른 값이 들어가지만, 임의로 제목과 동일하게 준 상태)

</p>

코드는 아래처럼 되어있었다.

```swift
    title: String = """
    아직 목표 시간을 채우지 않았어요.
    그래도 종료할까요?
    """,
    
    message: String = """
    아직 목표 시간을 채우지 않았어요.
    그래도 종료할까요?
    """,
    
    
private let titleLabel = UILabel().then {
  $0.numberOfLines = 0
  $0.textAlignment = .center
}

private let messageLabel = UILabel().then {
  $0.numberOfLines = 0
  $0.textAlignment = .center
}

titleLabel.attributedText = Typography.attributed(title, style: .headingLg, color: .appBlack)
messageLabel.attributedText = Typography.attributed(message, style: .bodyLg(weight: .regular), color: .gray500)
```

분명 `textAlignment`도 `.center`로 주었는데 해결이 되지 않아서 뭐가 무엇인지 찾아보았다.

<br>

## 문제 원인

찾아보니, `UILabel.textAlignment = .center`는 `plain text`일 때처럼 `label.text`를 직접 사용하는 경우에는 반영되지만, `attributedText`를 사용하면 그대로 반영되지 않는다고 한다.

정확히 얘기하면, `label.attributedText`를 사용하면 문자열 안에 이미 스타일 정보가 있다고 보고, `textAlignment`보다 `attributedText` 내부 `paragraphStyle`의 `alignment`값을 **우선 적용**한다.

<aside>

### 중간 간단 개념 정리

- **plain Text**
    - 그냥 문자열만 넣는 방식
    - 텍스트 내용만 존재. 문자열 구간별 스타일 정보는 없음
- **attributedText**
  - 문자열 + 스타일 정보 묶음
  - 폰트, 색, 줄간격, 정렬, 특정 단어만 볼드 처리 등을 문자열 안에 같이 담음
</aside>

<br>

그래서 테스트를 해봤다. 

위의 코드에서 `attributedText`가 아닌, `plain Text`를 사용했더니 아래 이미지처럼 정말로 중앙 정렬이 되었다.

## 내부 코드 분석

이번 프로젝트에서는 팀원분이 폰트, 문단 스타일 등 공통 적용할 수 있도록 `Typography.attributed(...)` 코드를 작성해두셨고, 해당 Alert에서도 그 코드를 사용하고 있었다.

그러보니 내부는 어떻게 되어있나 다시 코드를 살펴보니, 이미 내부에서 `paragraphStyle`을 만들어서 넣고 있었고, 정렬값이 아래처럼 `.natural`로 들어가고 있었다.

```swift
// 문단 스타일
static func paragraphStyle(for style: Style, font: UIFont? = nil) -> NSMutableParagraphStyle {
  let p = NSMutableParagraphStyle()
  let spec = spec(for: style)
  p.minimumLineHeight = spec.lineHeight
  p.maximumLineHeight = spec.lineHeight
  p.lineBreakMode = .byTruncatingTail
  p.alignment = .natural
  return p
}
```

즉, 중앙 정렬이 되지 않았던 건, `attributedText` 내부 기본 정렬 값이 `.natural`로 들어가 있기에 `textAlignment`보다 우선시 되어서 중앙 정렬이 되지 않았던 것이다.

이 부분을 `.center`로 수정해줄까 했지만, `Typography`는 공통으로 사용되는 코드라서, 이 값을 수정하면 다른 화면들까지 영향을 받을 수 있었다.

그래서 공용 코드는 그대로 두고, 지금 이 Alert에서만 중앙 정렬이 필요한거니까 이 부분에서만 따로 **중앙 정렬 헬퍼를 만들어 적용**하기로 했다.

<br>

## 해결

> **1. 헬퍼 추가**

```swift
private func centered(_ attr: NSAttributedString) -> NSAttributedString {
  let m = NSMutableAttributedString(attributedString: attr) // NSAttributedString(불변) -> NSMutableAttributedString(가변)으로 복사. 가변으로 바꿔야 속성을 추가/수정이 가능하기 때문
  let p = NSMutableParagraphStyle() // 문단 단위 속성(정렬, 줄간격, 들여쓰기, 줄바꿈 등)
  p.alignment = .center // 문단을 가운데 정렬
  p.lineBreakMode = .byWordWrapping // 줄바꿈을 "단어 단위 줄바꿈"으로.
  m.addAttribute( // NSMutableAttributedString에 단일 속성을 추가/갱신
    .paragraphStyle, // 어떤 속성을 추가/갱신할지. 문단 스타일 전체
    value: p, // 그 속성에 적용할 구체적인 값. (위에서 만든 p를 적용)
    range: NSRange(location: 0, length: m.length // 속성을 적용할 문자열 범위: 전체 텍스트
  ))
  return m // 정렬이 적용된 NSMutableAttributedString을 반환
}
```

위의 코드처럼, 중앙 정렬을 적용해주는 `centered` 헬퍼 코드를 작성했다.

자세한건 주석으로 설명을 작성해뒀지만, 간단히 말하면 기존 `NSAttributedString`을 가변 상태로 복사한 뒤, 가운데 정렬과 추가로 줄바꿈 설정을 문자열 전체에 적용하여 반환해주는 역할이다.

> **2. 코드 수정**

```swift
// 기존의 코드
titleLabel.attributedText = Typography.attributed(title, style: .headingLg, color: .appBlack)
messageLabel.attributedText = Typography.attributed(message, style: .bodyLg(weight: .regular), color: .gray500)

// 수정한 코드
titleLabel.attributedText = centered(Typography.attributed(title, style: .headingLg, color: .appBlack))
messageLabel.attributedText = centered(Typography.attributed(message, style: .bodyLg(weight: .regular), color: .gray500))
```

헬퍼가 `NSAttributedString`을 받아서 속성을 변경해주고, 반환값도 동일하게 `NSAttributedString`이기 때문에, 기존의 `attributedText` 값을 `centered` 헬퍼로 감싸주기만 하면 된다.
