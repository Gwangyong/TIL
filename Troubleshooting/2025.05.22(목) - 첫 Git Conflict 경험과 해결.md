- 아래와 같은 형태로 나온다고 했는데, 진짜 그랬네.. 위에가 내 코드고, 아래가 다른 팀원분이 작성하여 merge한 코드이다.

```swift
<<<<<<< HEAD
// 현재 브랜치의 코드 (Feat/#12-MoriVC-UI)
=======
 // develop 브랜치의 코드
>>>>>>> develop
```

- 이 중에 하나만 살리거나, 둘 다 조합해서 새로운 코드로 수정하면 된다고한다.

위처럼 중복된 것 중에 팀원분이 작성하신대로 동일하게 주석남긴걸로 작성하였다.

그리고 git에 다시 commit

```swift
git add theLastPush/theLastPush/Screens/Main/MainVC.swift

or 

git add .

git commit -m "fix: resolve merge conflict in MainVC.swift"
아래는 내가 작성한 커밋 메시지
git commit -m "fix: MainVC.swift에서 merge conflict 해결"
```


### **✅ 진행 한 순서**

1. ✅ git checkout develop
2. ✅ git pull origin develop (최신 develop 내용 가져오기)
3. ✅ git checkout Feat/#12-MoriVC-UI
4. ✅ git merge develop → 충돌 발생 ❗️
5. ✅ 충돌 파일 수동 해결 (MainVC.swift)
6. ✅ git add .
7. ✅ git commit -m "Fix: MainVC.swift에서 merge conflict 해결"
8. ✅ git status 확인 → working tree clean

- conflict가 어떤건지, 어떻게 해야해야하는지는 들어봤지만, 막상 간단한거라도 문제 터지니까 멈칫하는..
- 오히려 간단한 문제로 conflict를 해결하는 경험을 해봐서 나중에 더 큰 conflict가 발생하더라도
- 어떤식으로 해야해야하는지 알 수 있게되서 오히려 좋은 기회였다
