## 결론

타이머 화면 표시에서 기존에 경과 시간과 남은 시간을 `rounded()`로 같은 방식으로 처리했지만, 경과 시간은 `floor`, 남은 시간은 `ceil`을 적용하기로 변경했다.

<br>

## 왜 나눴는지

처음에는 경과 시간과 남은 시간을 하나의 포맷터로 처리했고, `Double` 값을 정수 초로 바꾸기 위해 `rounded()`를 사용했다.

- 기존 `rounded()` 사용 코드
    
    ```swift
      /// 시간 포맷터 ("h:mm:ss")
      private static func formatHMMSS(seconds: Double) -> String {
        let sec = Int(seconds.rounded())
        let h = sec / 3600
        let m = (sec % 3600) / 60
        let s = sec % 60
        return String(format: "%d:%02d:%02d", h, m, s) // 0:12:53, 1:50:49, 12:49:39등으로 포맷
      }
      
      /// 시간 포맷터 ("mm:ss")
      private static func formatMMSS(seconds: Double) -> String {
        let sec = Int(seconds.rounded())
        // 값이 3600이 들어옴
        // 이 3600을 60으로 나눠서(/) 그 값을 포맷팅
        let m = sec / 60
        let s = sec % 60
        return String(format: "%02d:%02d", m, s)
      }
    ```
    

그런데 실제로 다시 보니 경과 시간과 남은 시간은 사용자가 기대하는 표시 방식이 달랐다.

경과 시간은 **이미 지난 시간**을 보여주는 값이라, 아직 다 지나지 않은 초가 먼저 올라가면 어색했고, 반대로 남은 시간은 아직 **남아 있는 시간**을 보여주는 값이라, 아직 남아 있는 초가 먼저 사라지면 더 어색하게 느껴졌다.

결국 두 값을 같은 기준으로 처리하면 자연스럽게 보이기 어렵다고 판단했다.

<br>

## 예시 설명

> **경과 시간(올라가는 타이머) → `floor`(내림)**
> 

예를 들어 실제 경과 시간이 `10.1초`라면 화면에는 아직 `10초`로 보이는 쪽이 맞다. `10.9초`도 마찬가지다. 아직 `11초가 지난 것`은 아니기 때문이다.

- `10.1초 -> 10초`
- `10.9초 -> 10초`
- `11.0초 -> 11초`
- 즉, **“11초가 완전히 되기 전까지 10초”** 로 보이도록 유지

> **남은 시간(내려가는 타이머) → `ceil`(올림)**
> 

예를 들어 실제 남은 시간이 `10.9초`라면 화면에는 아직 `11초`가 남아 있는 것처럼 보여야 자연스럽다. `10.1초`도 마찬가지다. 이걸 바로 `10초`로 보여주면 아직 1초가 완전히 지나지 않았는데도 시간이 먼저 줄어든 느낌이 들기 때문이다.

- `10.9초 -> 11초`
- `10.1초 -> 11초`
- `10.0초 -> 10초`
- `9.9초 -> 10초`
- 즉, **“10초 아래로 내려가기 전까지는 11초 유지”** 가 되어야 사용자 입장에서 1초 빨리 깎이는 느낌이 들지않음

<br>

## Podoit 화면에서 더 중요했던 이유

TimerRun 화면은 **경과 시간**과 **남은 시간**이 같은 화면 안에 함께 보인다.

그래서 두 값을 같은 방식으로 처리하면 더 어색해질 수 있었다. 예를 들어 실제 값이 `경과 0.1초 / 남은 29.9초`일 때 둘 다 내림처럼 처리하면 화면에는 **`경과 0초 / 남은 29초`** 처럼 보일 수 있다.

사용자 입장에서는 시간이 서로 맞지 않게 흘러가서 이상하게 느낄 수 있다고 판단했다.

반대로 경과 시간은 `floor`, 남은 시간은 `ceil`로 나누면 같은 상황에서 `경과 0초 / 남은 30초`로 보여서 훨씬 자연스럽다.

<br>

## 적용

그래서 최종적으로는 **경과 시간용 포맷터**와 **남은 시간용 포맷터**로 분리했다.

```swift
/// 경과 시간 포맷터 ("h:mm:ss")
private static func elapsedFormatHMMSS(seconds: Double) -> String {
  let sec = Int(floor(seconds))
  let h = sec / 3600
  let m = (sec % 3600) / 60
  let s = sec % 60
  return String(format: "%d:%02d:%02d", h, m, s)
}

/// 경과 시간 포맷터 ("mm:ss")
private static func elapsedFormatMMSS(seconds: Double) -> String {
  let sec = Int(floor(seconds))
  let m = sec / 60
  let s = sec % 60
  return String(format: "%02d:%02d", m, s)
}

/// 남은 시간 포맷터 ("mm:ss")
private static func remainingFormatMMSS(seconds: Double) -> String {
  let sec = Int(ceil(seconds))
  let m = sec / 60
  let s = sec % 60
  return String(format: "%02d:%02d", m, s)
}
```
